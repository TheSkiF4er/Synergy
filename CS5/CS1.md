# Аналитический обзор реализации программы `birthdate_app.py`

## 1. Функциональность

### Оценка

Программа корректно реализует заявленные требования:

* Получение даты рождения от пользователя (день, месяц, год);
* Определение дня недели с использованием модуля `datetime`;
* Проверка високосного года по корректному алгоритму:

  ```
  year % 4 == 0 and (year % 100 != 0 or year % 400 == 0)
  ```
* Расчёт текущего возраста с учётом наступления дня рождения в текущем году;
* Вывод даты в формате `дд мм гггг`;
* Стилизованное отображение цифр через ASCII-графику.

### Выявленные ограничения

* Отсутствует строгая валидация диапазонов (день 1–31, месяц 1–12);
* Некорректная дата (например 31.02.2000) обрабатывается через исключение, но без повторного ввода;
* Нет защиты от логически невозможных дат (будущие даты рождения).

### Рекомендации

* Добавить явную проверку диапазонов перед созданием объекта `date`;
* Реализовать цикл повторного ввода до получения корректной даты;
* Добавить проверку `birth_date <= today`.

Производительность высокая.

* Используются стандартные библиотечные операции;
* Временная сложность — O(1);

Для CLI-приложения с единичным вычислением производительность избыточна.

### Потенциальные узкие места

Отсутствуют в текущем масштабе задачи.

### Рекомендации

* Перенести логику в отдельный модуль;
* Использовать пакетную обработку;
* Добавить профилирование при расширении функционала.

   * дата в будущем → ошибка

3. Проверка граничных значений:

   * 29.02.2024
   * 31.04.2020 (ошибка)
